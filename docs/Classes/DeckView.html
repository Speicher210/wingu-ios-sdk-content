DeckView

public class DeckView : UIView

Representation of Content (Deck) configured in wingu portal
It’s a view that usually is used inside whole UIViewController. It shows all supported components with current version of wingu-ios-sdk-essentials
If there is a need to get callbacks from this view protocol is available.

  • DeckAction

    Deck action represents all action that can be performed in a deck. Please note that action supported here not necessery match content configured in wingu portal, but it’s highly recommended to support all the cases since wingu content can be updated without app updaete.
    It can be used always with performDefault(_:payload:userInfo:navController:) method in DeckView object.

  • drawDeck(_:configuration:composition:)

    Main channel of communication between model and view.
    All components can be drawn here based on Channel and WinguComposition.

    Note: By default composition is set to defaultComposition it’s recommended to override some properties there to match your company identity and branding. To lear more visit WinguComposition struct.
