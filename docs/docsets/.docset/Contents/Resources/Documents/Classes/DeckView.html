DeckView

public class DeckView : UIView

The representation of Content (Deck) as configured in the wingu portal
It is a view that is usually used inside the entire UIViewController. It shows all supported components within the current version of wingu-ios-sdk-essentials
For this view a protocol is available when callbacks are needed.

  • DeckAction

    Deck action represents all action that can be performed in a deck. Please note that action supported here does not necessarily match content configured in the wingu portal. However, it is highly recommended that all cases are supported, since wingu content can be updated without requiring an app update.
    It can be used always with performDefault(_:payload:userInfo:navController:) method in DeckView object.

  • previewDeck(_:configuration:composition:)

    The previewDeck allows you to see a preview of the deck contents.
    However, it will have a limited view and only show and load to memory the first 1-3 components.
    It’s mostly use to render preview of an available Content with limit system resources and time.

    Note: If you invalidate the “parent view,” the deck needs to be drawn again
